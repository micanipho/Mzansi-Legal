# Implementation Plan: Full UI Pages вЂ” MzansiLegal Design System

**Branch**: `feat/019-full-ui-pages` | **Date**: 2026-03-31 | **Spec**: [spec.md](../../feat-019-full-ui-pages/spec.md)
**Input**: Feature specification from `specs/feat-019-full-ui-pages/spec.md`

## Summary

Implement and polish all eight primary page routes of the MzansiLegal Next.js 16.2 frontend in accordance with the approved visual design system. The pages (Home, Ask/Chat, Contracts List, Contract Detail, My Rights, History, Auth, Admin Dashboard) already have foundational implementations that must be restored to the working tree and enhanced to match the design references exactly вЂ” applying the established design tokens (CSS variables, Fraunces serif + Nunito sans, organic border radii, olive-green palette) consistently across all pages and components.

## Technical Context

**Language/Version**: TypeScript 5.x / Next.js 16.2 (App Router, `[locale]` i18n segment)
**Primary Dependencies**: Ant Design 6.x, next-intl 4.x, lucide-react, antd-style createStyles (already installed вЂ” no new packages)
**Storage**: JWT tokens in cookies (`ml_token`, `ml_user`); no new database tables
**Testing**: Manual acceptance tests per user story; Next.js dev server
**Target Platform**: Desktop (1280px+) and tablet (768px) web browsers
**Project Type**: Web application вЂ” frontend only (no backend changes)
**Performance Goals**: First AI answer content visible within 3 seconds; language switch < 1 second
**Constraints**: No new npm packages; mobile (<768px) layout out of scope for this iteration
**Scale/Scope**: 8 routes, ~15 components, 4 languages

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- [x] **Layer Gate**: Frontend-only feature вЂ” no new backend entities. N/A for DDD layer rules.
- [x] **Naming Gate**: No new backend services or controllers. All component/hook names follow existing conventions (PascalCase components, camelCase hooks, `use`-prefix hooks).
- [x] **Coding Standards Gate**: TypeScript throughout; file-based routing; no SSR on pages that don't need real-time data; Next.js `Image` component for all images; global state kept minimal.
- [x] **Skill Gate**: Applicable skills identified вЂ” `add-list-page`, `add-detail-page`, `add-styles`, `add-styling`, `add-auth-provider`, `add-service`, `follow-git-workflow`.
- [x] **Multilingual Gate**: All user-facing strings use `next-intl` translation keys; four language files (en.json, zu.json, st.json, af.json) will be kept in sync for all new/updated strings.
- [x] **Citation Gate**: Ask page renders citation references from RAG backend response; citations display act name + section number. Display contract is documented in `/contracts/qa-display.md`.
- [x] **Accessibility Gate**: All interactive elements include ARIA labels and support keyboard navigation; expandable cards use `aria-expanded`; language selector is keyboard-accessible.
- [x] **ETL/Ingestion Gate**: Not applicable вЂ” this feature adds no document ingestion functionality.
