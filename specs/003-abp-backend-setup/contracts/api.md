# API Contracts: ABP Backend Foundation Setup

**Feature**: 003-abp-backend-setup
**Phase**: 1 — Design

---

## Overview

This feature does not introduce new application endpoints. The API surface is the ABP Zero
built-in baseline. All endpoints below are auto-generated by the ABP framework.

---

## Built-in ABP Zero Endpoints

### Authentication

```
POST /api/TokenAuth/Authenticate
```

**Request**:
```json
{
  "userNameOrEmailAddress": "admin",
  "password": "123qwe",
  "rememberClient": false
}
```

**Response** (200 OK):
```json
{
  "accessToken": "<jwt>",
  "encryptedAccessToken": "<encrypted>",
  "expireInSeconds": 86400,
  "userId": 1
}
```

---

### Health / Liveness

```
GET /api/HealthCheck/Get
```

**Response** (200 OK):
```json
{
  "success": true
}
```

---

### Swagger UI

```
GET /swagger
```

Returns the Swagger UI HTML page listing all registered endpoints.

```
GET /swagger/v1/swagger.json
```

Returns the OpenAPI 3.0 JSON document.

---

## Notes

- All endpoints require HTTPS on `https://localhost:44311` in development.
- JWT token from `/api/TokenAuth/Authenticate` must be supplied as
  `Authorization: Bearer <token>` on protected endpoints.
- The Swagger UI provides an **Authorize** button for token entry.
